# CT-LIO 使用说明

CT-LIO 是一套基于连续时间 LiDAR-IMU 紧耦合里程计系统（Continuous-Time LiDAR-Inertial Odometry），支持 Robosense、Velodyne、Ouster、Livox 等多种雷达。本文档说明如何使用 Docker 环境运行系统。

---

## 目录结构

```
ct-lio/
├── bags/               # 存放 rosbag 数据（不打包进镜像）
├── config/
│   └── mapping.yaml    # 算法参数配置（挂载进容器，改后无需重建镜像）
├── docker/
│   ├── entrypoint.sh
│   └── patch_cmake.py
├── docs/               # 本文档所在目录
├── launch/
│   ├── run_eskf.launch
│   └── rviz.rviz
├── src/                # C++ 源码
├── Dockerfile
└── docker-compose.yml
```

## 首次构建镜像

```bash
cd /home/mingren/work/3DVision/ct-lio
docker compose build
```

构建时间约 10–20 分钟（含 Ceres 源码编译）。构建完成后镜像名为 `ct-lio:latest`。

> **仅在以下情况需要重新构建：**
> - 修改了 `src/` 下的 C++ 源码
> - 修改了 `Dockerfile` 或 `CMakeLists.txt`

## 准备数据

将 rosbag 文件放入 `bags/` 目录：

```bash
cp /your/path/xxx.bag bags/
```

容器内对应路径为 `/bags/`。

---

## 运行系统（推荐：四终端分窗）

### 第一步：允许容器访问显示器

在宿主机执行一次（每次重启系统后需重新执行）：

```bash
xhost +local:docker
```

### 第二步：启动容器

```bash
cd /home/mingren/work/3DVision/ct-lio
docker compose run --name ct-lio-run --rm ct-lio bash
```

在容器内启动 ROS 核心：

```bash
roscore
```

### 第三步：启动算法节点（新终端）

```bash
docker exec -it ct-lio-run bash
roslaunch ct_lio run_eskf.launch
```

### 第四步：打开 RViz 可视化（新终端）

```bash
docker exec -it ct-lio-run bash
rviz -d /catkin_ws/src/ct-lio/launch/rviz.rviz
```

### 第五步：播放数据（新终端）

```bash
docker exec -it ct-lio-run bash
rosbag play /bags/2022-08-30-20-33-52_0.bag
```

> **播放控制：**
> - 空格键：暂停 / 继续
> - `s`：单帧步进
> - `-r 0.5`：以 0.5 倍速播放，如 `rosbag play /bags/xxx.bag -r 0.5`

## 参数配置（`config/mapping.yaml`）

配置文件通过 volume 挂载进容器，**修改后无需重建镜像，重启算法节点即可生效**。

### 关键参数说明

```yaml
preprocess:
  lidar_type: 4          # 1=Livox AVIA, 2=Velodyne, 3=Ouster, 4=Robosense, 5=Pandar
  point_filter_num: 1    # 每 N 个点取 1 个（预过滤）
  blind: 0.01            # 剔除雷达原点附近 blind 米内的点

common:
  imu_topic: /imu/data        # IMU ROS topic 名称
  lid_topic: /rslidar_points  # LiDAR ROS topic 名称

mapping:
  extrinsic_est_en: true      # true = 在线估计外参，false = 使用固定外参
  extrinsic_T: [0.065, 0.0, -0.05]          # LiDAR→IMU 平移（米）
  extrinsic_R: [1,0,0, 0,-1,0, 0,0,-1]     # LiDAR→IMU 旋转矩阵（行优先）

odometry:
  surf_res: 0.2               # 降采样体素大小（米），越小点越密
  max_num_iteration: 15       # ICP 外层最大迭代次数
  size_voxel_map: 0.2         # 地图体素大小（米）
  max_dist_to_plane_icp: 0.1  # 点到平面最大匹配距离（米）
  max_num_residuals: 3000     # 每帧最多使用残差数
  min_num_residuals: 300      # 少于此数则跳过本帧
  motion_compensation: CONSTANT_VELOCITY  # 运动补偿模式（见下方说明）
```

### `motion_compensation` 选项

| 值 | 说明 |
|----|------|
| `NONE` | 不做运动补偿 |
| `CONSTANT_VELOCITY` | 假设匀速运动（不依赖 IMU） |
| `ITERATIVE` | 迭代式补偿 |
| `CONTINUOUS` | 连续时间补偿（精度最高，依赖 IMU） |

## 适配新数据集

1. **确认 ROS topic 名称：**

   ```bash
   rosbag info /bags/your.bag
   ```

2. **修改 `config/mapping.yaml`：**

   ```yaml
   common:
     imu_topic: /your/imu_topic
     lid_topic: /your/lidar_topic
   preprocess:
     lidar_type: 2   # 按实际雷达类型修改
   ```

3. **修改外参：** 根据实际传感器安装位置填写 `extrinsic_T` 和 `extrinsic_R`。

4. **时间戳模式（`src/apps/main_eskf.cpp` 第 82 行）：**

   ```cpp
   // 正常数据集（header 时间戳为帧起始）
   lio->pushData(cloud_out, std::make_pair(msg->header.stamp.toSec(), convert->getTimeSpan()));

   // 部分数据集（header 时间戳为帧结束，如 staircase）
   lio->pushData(cloud_out, std::make_pair(msg->header.stamp.toSec() - convert->getTimeSpan(), convert->getTimeSpan()));
   ```

   修改此行需要重新构建镜像。

---

## 注意事项

- **外参 `extrinsic_R` 非常关键。** 若旋转矩阵错误，IMU 轴向与 LiDAR 不一致，轨迹会严重漂移。始终以传感器手册或标定结果为准。

- **`bags/` 目录只读挂载进容器**，容器内无法向其写入文件。

- **`config/` 目录只读挂载进容器**，若需在容器内临时修改，可先 `cp` 到 `/tmp/` 再手动加载。

- **ICP 退化警告 `Low convincing result`：** 出现在几何特征不足的场景（长走廊、空旷区域）。少量（< 10%）属正常现象；大量出现说明外参或参数需要调整。

- 修改 `src/` 下任何 `.cpp`/`.h` 文件后，必须重新 `docker compose build` 才能生效。

- 若 RViz 无法弹出窗口，检查宿主机是否执行了 `xhost +local:docker`，以及 `DISPLAY` 环境变量是否正确。

## 快速参考命令

```bash
# 构建镜像
docker compose build

# 启动容器（交互式）
docker compose run --name ct-lio-run --rm ct-lio bash

# 进入已运行的容器
docker exec -it ct-lio-run bash

# 停止并删除容器
docker rm -f ct-lio-run

# 查看算法日志
docker exec ct-lio-run cat /tmp/launch.log

# 统计退化帧数量
docker exec ct-lio-run grep -c 'Low convincing' /tmp/launch.log
```
